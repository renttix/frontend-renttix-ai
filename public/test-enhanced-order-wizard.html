ðŸ§ª Test Scenarios

Scenario 1: Repeat Customer with Autofill

Objective: Test customer intelligence and autofill functionality

  1. Select a customer who has previous orders
  2. Verify the customer type badge appears correctly
  3. Check that autofill suggestions appear
  4. Confirm previous order details are populated
  5. Verify smart duration suggestion based on history
// Expected behavior: - Customer badge shows "Account" or "Prepay" - Previous delivery address auto-populates - Common products are suggested - Average rental duration is suggested

Scenario 2: Asset Conflict Resolution

Objective: Test real-time availability and conflict detection

  1. Add products to the order
  2. Select dates that overlap with existing bookings
  3. Verify conflict warnings appear
  4. Use bulk assignment to find available alternatives
  5. Confirm asset conditions are displayed
// Test data: Product: "6ft Fence Panel" Dates: Select dates with known conflicts Expected: Conflict alert with resolution options

Scenario 3: Recurring Delivery Configuration

Objective: Test recurring delivery setup and preview

  1. Navigate to Delivery Magic step
  2. Enable recurring delivery
  3. Configure weekly pattern
  4. Set end date or number of occurrences
  5. Preview upcoming delivery dates
  6. Verify route cost estimation updates
// Configuration example: Pattern: Weekly Frequency: Every 2 weeks End: After 10 deliveries Expected: Preview shows 10 future dates

Scenario 4: Address Validation & Route Estimation

Objective: Test Google Places integration and route costing

  1. Start typing a partial address
  2. Select from autocomplete suggestions
  3. Verify address is validated (green checkmark)
  4. Check route cost estimation appears
  5. Verify delivery windows are updated
  6. Test manual address entry fallback
// Test addresses: "10 Downing Street, London" "Buckingham Palace" "Invalid Address XYZ" (test fallback)

Scenario 5: File Attachments & Validation

Objective: Test file upload and validation warnings

  1. Navigate to Confirm & Go step
  2. Drag and drop files to attachment zone
  3. Verify file preview for images/PDFs
  4. Test file size limits (>10MB should fail)
  5. Check validation warnings appear
  6. Acknowledge warnings before submission
// Test files: - Small PDF (< 1MB) - Large image (> 10MB) - should fail - Multiple files drag & drop - Unsupported format (.exe) - should reject
